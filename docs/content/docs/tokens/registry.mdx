## Token Naming Convention

| Prefix | Meaning | Example |
|--------|---------|---------|
| `L*` | Liquid token on Lux | LETH (1:1 for deposited ETH) |
| `Z*` | Liquid token on Zoo | ZETH (1:1 for deposited ETH) |
| `x*` | Liquid staking | xLUX, xZOO (yield vault shares) |
| `v*` | Vote-escrowed | vLUX (xLUX+DLUX), vZOO (xZOO+KEEPER) |
| `s*` | Validator staked | sLUX |
| `W*` | Wrapped native | WLUX, WZOO |
| `K` | Karma | Soul-bound reputation (DID-linked) |

## Contract Implementations

All liquid tokens (L* and Z* prefixes) use the [LRC20B](/docs/tokens/lrc20b) implementation with teleport minting:

```solidity
import "@luxfi/standard/src/tokens/ERC20B.sol";

// LETH deployment
contract LuxETH is ERC20B {
    constructor() ERC20B("Lux ETH", "LETH") {}
}
```
